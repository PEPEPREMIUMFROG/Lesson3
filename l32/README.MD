
# Network Management System

Консольное приложение для учёта и управления сетевыми объектами: сетями, устройствами и подключениями между ними.

Реализовано в соответствии с требованиями учебного проекта по многоуровневой архитектуре (UI → Service → DAO → БД).

---

## Функциональность

- **Сети**: добавление, редактирование, удаление (только пустых сетей)
- **Устройства**: привязка к сети, валидация IP/MAC, поиск по IP/типу/статусу
- **Подключения**: связь между двумя устройствами (даже из разных сетей)
- **Отчёты**: статистика по сети (всего устройств, активных, разбивка по типам)
- **Поиск**: гибкая фильтрация сетей и устройств

---

## Запуск приложения

### Вариант 1: Через Maven (рекомендуется)

Приложение **автоматически создаёт структуру БД** при запуске.

>  Для работы требуется **Docker** (используется Testcontainers).

```bash
mvn compile exec:java "-Dexec.mainClass=org.example.Main"
```

> При первом запуске будет автоматически поднят временный PostgreSQL-контейнер.  
> Нет необходимости вручную создавать БД или пользователя.

---

### Вариант 2: Вручную (если Docker недоступен)

1. Установите **PostgreSQL 15+**
2. Создайте БД и пользователя:
   ```sql
   CREATE DATABASE networks_db;
   CREATE USER admin WITH PASSWORD 'admin';
   GRANT ALL PRIVILEGES ON DATABASE networks_db TO admin;
   ```
3. Запустите:
   ```bash
   mvn compile exec:java "-Dexec.mainClass=org.example.Main"
   ```

## ⚠️ Важно

- Устройства проходят **валидацию**:
    - IP: формат `xxx.xxx.xxx.xxx` (например, `192.168.1.1`)
    - MAC: формат `XX:XX:XX:XX:XX:XX` (например, `00:1B:44:11:3A:B7`)
- Удалить сеть можно **только если в ней нет устройств**
- Подключение удаляется автоматически при удалении любого из устройств
