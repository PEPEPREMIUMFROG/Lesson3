
# Тестирование Network Management System

## 1. Модульное тестирование (Unit Tests)

Проект содержит модульные тесты, написанные с использованием **JUnit 5** и **Mockito**.  
Тесты проверяют корректность бизнес-логики без обращения к реальной базе данных.

### Покрытые сценарии:
- Валидация входных данных (IP-адрес, MAC-адрес) — через `ValidationUtil`.
- Логика сервиса:
    - Невозможно добавить устройство, если нет сетей.
    - Поиск сетей по имени.
    - Поиск устройств по критериям (IP, тип, статус).
    - Невозможно удалить сеть, если в ней есть устройства (проверяется на уровне UI/сервиса).
- Все зависимости DAO замокированы (`@Mock`), т.е. тестируется **только бизнес-логика**.

Запуск:
```bash
mvn test
```

## 2. Интеграционное тестирование (Integration Tests)

Интеграционные тесты используют **Testcontainers** для запуска временного экземпляра PostgreSQL в Docker-контейнере.

### Покрытые сценарии:
- Создание сетей и устройств в реальной БД.
- Сохранение и извлечение данных через `NetworkDao`.
- Поиск по имени и фильтрация устройств.
- Генерация статистики по сети (количество устройств, активные, разбивка по типам).

Запуск:
```bash
mvn verify
```
(интеграционные тесты запускаются на фазе `verify`)

## 3. Ручное тестирование

Приложение можно запустить вручную для проверки сквозного сценария:

```bash
mvn compile exec:java "-Dexec.mainClass=org.example.Main"
```

Поддерживаемые сценарии:
- Добавление/редактирование/удаление сетей.
- Добавление устройств с валидацией IP/MAC.
- Создание подключений между устройствами.
- Просмотр деталей сети и статистики.
- Генерация отчёта по всем сетям.

> ⚠️ Для запуска вручную требуется локальный PostgreSQL с параметрами из `application.properties`,  
> **либо** запуск через Docker:
> ```bash
> docker run --name networks-db -e POSTGRES_DB=networks_db -e POSTGRES_USER=admin -e POSTGRES_PASSWORD=admin -p 5432:5432 -d postgres:15
> ```

## 4. Валидация данных

При добавлении или редактировании устройства автоматически проверяются:
- Формат IP-адреса (например, `192.168.1.1` — валидный, `999.999.999.999` — нет).
- Формат MAC-адреса (например, `00:1B:44:11:3A:B7` — валидный, `invalid-mac` — нет).

При нарушении формата выводится ошибка:  
`Invalid IP address format` или `Invalid MAC address format`.
